# Product

## Register

product

## Users

Working developers practicing and skilling up on the side. They know their way around an editor, expect a real dev tool (not a toy), and value signal over decoration. They open it between real work — calm ambient focus, not hype. They want the session to feel like a sharpened instrument: fast, legible, respectful of their time and attention.

## Product Purpose

AgentX is a "Write → Sabotage → Fix" debugging trainer. Three LLM agents (Host, Saboteur, Evaluator) drive a loop: the Host presents a real coding challenge, the student writes a solution, the Saboteur injects subtle bugs, the student fixes them, and the Evaluator scores the round. The product exists to make developers measurably better at reading, reasoning about, and repairing broken code — with a feedback loop tight enough that skill is visible after one session.

## Brand Personality

Calm. Considered. Crafted.

The interface should feel like a well-made notebook for code: generous whitespace, quiet confidence, typographic discipline. No hype, no drama, no gamified reward bursts. Precision is the personality. The agents (Host / Saboteur / Evaluator) are present as distinct voices, but they are collaborators, not mascots.

## Anti-references

- **Generic SaaS dashboard** — cream/beige backgrounds, rounded cards everywhere, blue-purple gradients, the "AI tool" visual cliché.
- **Bootcamp tutorial site** — cheerful, hand-holdy, big cartoon illustrations, green checkmarks, "you got this!" energy.
- **Gamified kids' coding** — bright primary colors, badges, mascots, confetti. Reads as childish, not serious.
- **AI-generated look** — violet gradients, emoji-as-icons, glassmorphism, gradient text, hero-metric template. The full slop tell.

## Design Principles

1. **Instrument, not stage.** The editor and the trace are the product. Everything else gets out of their way. No surface competes with the code.
2. **Signal over noise.** Each pixel carries information about phase, score, or agent activity. Decoration earns its place by clarifying state.
3. **Quiet confidence.** Motion is rare and purposeful; color is restrained and semantic; typography does the hierarchy work. The interface trusts the user to read.
4. **One canvas, three voices.** Host, Saboteur, and Evaluator are distinguished by typography and a single accent role each — not by icons, mascots, or saturated color blocks.
5. **Respect the craft.** Spacing is deliberate, alignment is exact, copy is precise. The product is for developers who notice details; the details should reward them.

## Accessibility & Inclusion

WCAG AA target (4.5:1 body, 3:1 large text and UI boundaries). Reduced-motion respected on every transition. Color is never the sole carrier of phase or agent identity — typography and labels back it up. Editor remains keyboard-navigable; all actions reachable without a pointer.
